## 一、修复内容

### 1.1 threshold参数Bug修复

#### 问题描述
- 备用API的`/api/monitor/active`端点threshold参数不生效
- 请求传入`{"threshold": 60}`，但返回`"threshold": 30.0`

#### 根本原因
Railway上运行的代码使用Query参数而非Body参数，导致客户端JSON body中的threshold被忽略

#### 修复方案
```python
# ❌ 旧版本 (不生效)
@app.post("/api/monitor/active")
async def monitor_opportunities(
    threshold: float = Query(60.0, description="毛利门槛"),
    max_results: int = Query(3, description="最多处理数")
):

# ✅ 新版本 (正确生效)
@app.post("/api/monitor/active")
async def monitor_opportunities(
    threshold: float = Body(45.0, description="毛利门槛，默认45%"),
    max_results: int = Body(3, description="最多处理商机数")
):
```

### 1.2 毛利门槛调整

#### 原因
- 原门槛60%过高
- 当前数据源实际毛利分布为30%-50%
- 连续多日无60%+商机

#### 新门槛
- **45%** (原60%)

### 1.3 代码精简

将v3.2.0完整版简化为v2.4.0精简版：
- 减少启动时间
- 降低内存占用
- 提高稳定性
